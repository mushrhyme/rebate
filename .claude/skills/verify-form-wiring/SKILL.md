# verify-form-wiring 스킬

양식을 UI에서 동기화한 뒤, **블록·json·엔진까지 끝까지 연결됐는지** 결정적으로 점검한다.
진단에서 끝나지 않고 **safe 등급은 실제 파일을 고친다.**

## 언제

- 신규/수정 양식을 동기화한 직후 "제대로 붙었나" 확인할 때
- `[config]` 블록은 생겼는데 phase4·교차검증·식별까지 연결됐는지 의심될 때
- 배포 전 양식 정합성(`build --check`) 사전 점검

## 핵심 원칙 — 3갈래 라우팅 (자동수정 경계)

| 등급 | 의미 | 동작 |
|------|------|------|
| 🔧 **safe** | 승인된 정본에서 *기계적으로 재생성*(파생물). 정본 불가침·결정적·검증가능 | **자동수정** (`--fix`) |
| 👤 **owner** | 현업이 MD에 적고 동기화하면 반영될 규칙(TBD·식별패턴·미정의 변수) | **안내만** — 지어내지 않음 |
| 🛠 **dev** | 엔진에 없는 어휘(미등록 교차검증 type·미지원 연산·미등록 전략, T3) | **개발자에게 플래그** |

> **safe 4조건**(전부 충족): ① 정본에서 도출 가능 ② 정본 불가침 ③ 의미 불변 ④ 객관적 검증. 하나라도 깨지면 owner/dev로 내린다. 회계 규칙·엔진 코드는 절대 자동 추측·작성하지 않는다.

## 절차

### 1. 검증 실행

```bash
python scripts/verify_form_wiring.py {form_id}        # 특정 양식 (예: form_03)
python scripts/verify_form_wiring.py                  # 등록된 전 양식
```

엔진 어휘(교차검증 type·집계 전략·DSL 연산)는 **엔진 소스에서 직접 추출**해 대조한다(드리프트 없음).

점검 게이트: build↔json 동치 · 블록↔json 일치 · Schema · NET expr 어휘 · 교차검증 type 등록 · 집계 전략 등록 · 식별 패턴 · _index 등록 · 남은 TBD.

### 2. 결과 해석 + 라우팅

- **🔧 safe**: 사용자에게 한 줄로 알리고 `--fix` 적용을 제안한다.
- **👤 owner**: "현업이 MD에 ~를 채우고 동기화하면 반영됩니다"로 안내. **값을 지어내지 않는다.** TBD 항목·미정의 변수·식별 키워드 누락이 여기.
- **🛠 dev**: "이건 엔진 어휘에 없어 개발(T3)이 필요합니다 — 새 교차검증 종류/연산/전략"으로 플래그. 엔진 코드를 임의로 작성하지 않는다.

### 3. safe 등급 자동수정 (요청·확인 시)

```bash
python scripts/verify_form_wiring.py {form_id} --fix
```

safe 수정자만 적용한다:
- `config/form_types.json` 블록에서 재빌드(순서/드리프트 정정)
- `_index.md` 미등록 표기 → 등록행 채움(기계적 복사)

적용 후 재검증으로 게이트가 닫혔는지(`build 동치 True`) 확인해 보고한다.

### 4. 보고

```
[verify-form-wiring] form_03
✅ 연결: 블록↔json · schema · NET 어휘 · 교차검증 · 식별패턴
🔧 자동수정(적용): _index 등록행
👤 현업: 미확정 규칙 22개(TBD) — MD에 채우고 동기화
🛠 개발자: (없음)
```

## 제약

- **owner/dev 등급은 절대 파일을 고치지 않는다.** 회계 규칙·엔진 코드는 사람이 공급한다.
- safe 자동수정은 **파생물만** 건드린다(json은 블록에서 생성, _index는 form_XX.md에서 복사). 정본(블록·산문)은 불가침.
- 게이트·어휘 정의의 단일 출처는 `scripts/verify_form_wiring.py`. 백엔드 post-sync 훅도 동일 스크립트를 호출하면 드리프트가 없다.
