# カナカン株式会社 商品部食品課（北陸）— 旭食品/ヤマキ

- **약칭**: 旭食品/ヤマキ
- **수신처**: (株)農心ジャパン
- **메모**: 発行元はカナカン株式会社 商品部食品課（北陸）。口座コード `0-742-05534-51-20-742-2` が請求単位識別子として機能する。
- **특이사항**: 한 PDF에 복수 문서 묶임 — 請求書(cover) + 条件請求明細書(detail×N) + 請求確認書(summary/返信用) + 条件・経費請求NO一覧表 の順で1PDFに綴じられる。

## 식별 패턴

`条件請求明細書` · `請求確認書` · `カナカン株式会社` · `商品部食品課（北陸）` · `販促金`

---

## 문서 구조

```
PDF p.1      — cover        : 請求書（合計サマリー）
PDF p.2〜12  — detail       : 条件請求明細書（得意先別明細、複数ページ）
PDF p.13     — summary      : 請求確認書（返信用）
PDF p.14     — summary      : 条件・経費 請求NO一覧表
```

## 문서 내 계층 구조

```
口座 (0-742-05534-51-20-742-2)          — 추출 안 함
  └ 事業所 (例: 751 カナカン株式会社 食品富山営業所)  → items[].branch 로 상속
      └ 請求No (例: 3002129433)          → items[].invoice_no 로 상속
          └ 得意先名 (例: 島屋株式会社)  → items[].customer 로 상속
              └ 商品行                   ← 항목 추출 단위
```

## [Phase 2] OCR 정규화 규칙

### 数値列（金額・単価・数量）

| OCR 값       | 실제 값  | 해석                                              |
| ------------ | -------- | ------------------------------------------------- |
| `1,3440`     | `13440`  | 桁区切りカンマをOCRが小数点または位置ずれで誤読   |
| `2,280`      | `2280`   | 同上                                              |
| `3,8007511`  | `3800` / `7511` | 販売価格列と条件列が連結して読まれるケース |
| `14,0007511` | `14000` / `7511` | 同上                                             |

> 数値列は縦罫線が密集しているため、隣接する列の値が連結OCRされやすい。`7511` は条件列の固定値として頻出するため、末尾に `7511` が付いている場合は分離して処理する。
>
> **`条件` 列の小数処理**: OCR は整数として読むが、下2桁が小数部のため `÷100` して実値とする（例: `1850` → `18.50`、`7511` → `75.11`、`4460` → `44.60`）。

## [Phase 2] 함정 케이스

1. **小計行の除外**: `*** 小 計 ***` および `*** 税 込 計 ***` で始まる行は集計行のため item として抽出しない。
2. **税額行の除外**: `税額(軽減 8%)` と表示される行（例: `1,465`）は消費税集計行のため item として抽出しない。
3. **得意先名ブロックの継承**: 商品行の上に独立行として得意先名（例: `島屋株式会社`）が記載される。以降の商品行すべてに `customer` として引き継ぐ。次の得意先名行が現れるまで継続。
4. **担当者名行の除外**: 得意先名と同行または直後に記載される担当者名（例: `竹部保`、`沖崎暉`）は `columns["摘要"]` に格納し、item としては抽出しない。
5. **区分コード凡例**: 明細書最下部に `区分 1:ケース引き 2:%引き 3:口引き 4:儲引き 5:未収保証 7:ミッションダウン 8:現品添付通常入庫 9:現品添付景品入庫` と記載される。`columns["区"]` の値はこの凡例に対応する。
6. **複数事業所の混在**: 1PDFに751（食品富山営業所）等、複数の事業所が連続して記載される場合がある。事業所ヘッダー行をまたぐ場合は `branch` を都度更新して引き継ぐ。
7. **バラ数量のみの行**: `数量_ケース` が空欄で `数量_バラ` のみ記載される商品行がある（例: `農心 辛ラーメンミニカップ 49G` 行で `12` がバラ列）。空欄は `0` として処理する。
8. **請求NO一覧表ページ**: PDF p.14「条件・経費 請求NO一覧表」は支払い明細書としての利用を想定した集計表であり、item 抽出対象外とする（summary ロールで totals 照合にのみ使用）。
9. **摘要の継承スコープ**: `columns["摘要"]` の値は行ごとに記載されるケースと、同一請求No 内の商品行全体に継承されるケースが混在する。タイプ判定時は「当該行の摘要」と「請求No単位で継承された摘要」の両方を参照する。
10. **税率の取得**: 同一請求Noグループ末尾の `税額（軽減 8%）` 行の存否で当該グループの税率を判定する（8% グループは `税額（軽減 8%）` が存在する）。税率情報は行単位ではなく請求No単位で保持する。

- **단서 2**: 다음 페이지 상단に得意先名ブロックが存在せず商品行から始まっている → 직전 그룹의 연속
