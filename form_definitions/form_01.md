# form_01 — FINET 買掛金特別値引請求明細書

- **약칭**: FINET
- **발행처**: 卸売業者 (国分グループ 등 FINET 가맹 도매상)
- **수신처**: (株)農心ジャパン
- **특이사항**: 한 PDF = 단일 請求書 (請求書No. 1개)

## 식별 패턴

`得意先名` · `商品名` · `消費税率`

---

## 문서 구조

```
PDF 전체
  └ 請求書No. XXXXXX (단일)
      ├ Page 1      — cover (販促金請求 計 8%/10% + 役務提供 計)
      ├ Pages 2〜N-1 — detail (得意先별 請求伝票番号 그룹)
      └ Page N      — summary 또는 detail (得意先小計 목록 / 役務提供 명세)
```

---

## 문서 내 계층 구조

```
得意先名 (거래처)       → 각 항목에 기록
  └ 請求伝票番号 + 計上No → 이어붙여 항목 식별자로 기록
      └ ← 항목 추출 단위
```

| role        | 판단 기준                                                      |
| ----------- | -------------------------------------------------------------- |
| `cover`   | `販促金請求 計` · `役務提供 計` 합계 테이블만 있는 페이지 |

| `summary` | 得意先別 소계 목록만 있는 페이지                               |

| 키                              | 필수 | 타입 | 비고              |
| ------------------------------- | ---- | ---- | ----------------- |
| `販促金請求 8%対象 税抜`      | ✓   | 정수 |                   |
| `販促金請求 8%対象 消費税`    | ✓   | 정수 |                   |
| `販促金請求 10%対象 税抜`     | ✓   | 정수 |                   |
| `販促金請求 10%対象 消費税`   | ✓   | 정수 |                   |
| `販促金請求 非課税対象`       | —   | 정수 | 없으면 생략       |
| `販促金請求 今回請求金額合計` | ✓   | 정수 | 税込 합계         |
| `役務提供 今回請求金額合計`   | ✓   | 정수 | 없으면 0으로 기록 |

## [Phase 2] 함정 케이스

1. **小計 행 제외**: `請求伝票番号 小計` 행은 item 아님 → `items[]` 제외.
2. **役務提供 테이블**: cover 페이지의 `役務提供 計` 테이블은 `totals`에만 포함. item으로 추출하지 않는다.
3. **条件区分=円**: 条件이 고정금액. `columns["数量"]` = null, `columns["数量単位"]` = `"円"`. 条件 값이 곧 金額이며 수량 무관 (예: 農心 代表 ドライ — 32,220¥ 고정).
4. **条件区分=%**: 物流費補助 등 매출액 기준 수수료 타입. `columns["数量単位"]` = `"円"`, `columns["数量"]` = 매출액(¥), `columns["条件"]` = 비율(소수 가능, 예: `3.4`), 金額 = round(数量 × 条件%). 동일 得意先 + 商品名에서 複数の計上No로 나뉘어 등장 가능 (예: 0001·0002·0003이 각각 다른 매출 기반). 商品名 보조행의 `◆物流費補助` 등 레이블은 Phase 1 병합 → `columns["備考"]`에 포함. 伝票 끝에 `消費税金額 N%` 레이블 행이 나타나며 小計에 포함됨 — 0000000 패턴과 다른 형식. 추출 여부 TBD. NET 계산 TBD — 현업 확인 필요.
5. **OCR 중복 備考**: 주행의 備考에 `8.00% 外税`가 OCR 오류로 복사되는 경우 무시. 보조행 병합값만 사용.
6. **伝票番号 그룹**: 동일 請求伝票番号 아래 002, 003 행은 독립적인 item. 각각 별도 추출.
7. **消費税計上 항목**: 得意先名 = `YYYY.MM月度消費税計上 (0000000)` 형식. 商品名도 동일 텍스트. `items[]`에 포함. タイプ = `非課税`. 소매처·제품 매핑은 현업 확인 필요로 표시. 아래 두 패턴이 존재한다.
   - **独立 伝票型**: 消費税計上 行이 독립된 伝票番号를 가짐 (예: `2501000000` — 말미 all-zeros).
   - **同一 伝票型**: 消費税計上 行이 일반 item과 동일 伝票番号를 가지며 다른 計上No로 등장.

## [Phase 2] summary 페이지 추출

summary role 페이지(得意先別売上明細)에서 `customer_summaries` 키를 추출한다.

- **키**: 得意先名 원문 (괄호 코드 포함). 例: `"(株) アレスカンパニー (TP) (243514)"`
- **값**: 해당 得意先의 소계 금액 (쉼표 제거 정수, **税抜**)
- 두 가지 OCR 레이아웃이 혼재한다:
  - **레이아웃 A**: 得意先名 행에 金額이 있고, 다음 행이 `得意先小計` (금액 없음)
    → 得意先名 행의 金額을 소계로 사용
  - **레이아웃 B**: OCR 컬럼 합성으로 得意先名 + `得意先小計` + 金額이 한 행에 나타남
    → 해당 행의 金額을 소계로 사용

출력 예:
```json
{
  "page": 15,
  "role": "summary",
  "customer_summaries": {
    "(株) アレスカンパニー (TP) (243514)": 15456,
    "(株) 7-11 《集 (300691)": 15448195
  }
}
```

## [Phase 4] NET 계산식

Python 코드 전용. Phase 2·3에서는 이 섹션을 무시한다.

### NET 계산

| 조건              | 수식                                                                |
| ----------------- | ------------------------------------------------------------------- |
| `条件区分 = 個` | `NET = 仕切 - 条件`                                               |
| `条件区分 = CS` | `NET = 仕切 - (条件 / ケース入数)`                                |
| `条件区分 = 円` | NET 계산 없음. AL열(金額)만 채움                                    |
| `条件区分 = %`  | `金額 = round(数量(¥) × 条件%)`. NET 계산 TBD — 현업 확인 필요 |

### 교차검증

| #  | 좌변 (items 집계)       | 우변 (문서 값)                                                        | 비고                                  |
| -- | ----------------------- | --------------------------------------------------------------------- | ------------------------------------- |
| ① | detail 全 `金額` 합산 (税抜) | cover `販促金請求 8%対象 税抜` + `販促金請求 10%対象 税抜` 합산 | 消費税計上 行 처리는 현업 확인 필요 |
| ② | summary 得意先小計 합계 (税抜) | cover `販促金請求 8%対象 税抜` + `販促金請求 10%対象 税抜` 합산 | 得意先小計는 税抜. 今回請求金額合計(税込)와 비교하지 않는다 |
| ③ | 得意先별 `金額` 합산  | summary `得意先小計`                                                |                                       |
