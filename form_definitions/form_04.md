# form_04 — 日本アクセス 販売促進金請求書 (CVS)

- **약칭**: ACCESS
- **발행처**: 株式会社 日本アクセス (各支店 — 生鮮直轄·九州 등)
- **수신처**: (株)農心ジャパン
- **특이사항**: 한 PDF 안에 여러 請求書가 묶임 — 請求書No.별로 독립 처리 (cover 페이지 감지로 경계 판단)

## 식별 패턴

`得意先又は商品` · `未収条件`

## 문서 구조

```
PDF 전체
  ├ 請求書No.004796239 (8% 軽減税率)
  │   ├ Page 1    — cover (1/8)
  │   ├ Pages 2–8 — detail (2/8 ~ 8/8)
  │   └ Page 9    — payment_form (1/1) ← 무시
  ├ 請求書No.004796240 (10% 標準税率)
  │   ├ Page 10   — cover (1/2)
  │   ├ Page 11   — detail (2/2)
  │   └ (payment_form)
  └ … (이후 반복)
```

```

```

## [Phase 2] 条件タイプ 정의

일본 아쿠세스 청구서에서는 동일 제품에 대해 복수의 管理No가 개별 조건 단위로 분리되어 기재될 수 있다.

| 条件タイプ       | 의미                                                        | 수량 관계                          |
| ---------------- | ----------------------------------------------------------- | ---------------------------------- |
| `定番条件`     | 전체 판매수량에 적용되는 기본 조건                          | 기준 판매수량                      |
| `原価引き条件` | 定番 수量 중 일부에 추가 적용되는 원가 인하 조건 (이중조건) | 原価引き 수量 ≤ 対応する定番 수量 |
| `導入条件`     | 신규 입점·진열 시 定番과 동일 수량에 추가 적용되는 조건    | 導入 수量 = 対応する定番 수量      |

**업무 맥락:**

■ **定番条件**
기본 판매 조건이며 실제 판매수량의 기준이 된다.

■ **原価引き条件**
定番조건과 별개의 판매가 아니라, 定番 판매수量 중 일부 물량에 추가 차감이 적용되는 구조다.

즉:

- 定番 수량 전체에 기본 조건 적용
- 그 중 일부 수량에만 原価引き 조건 추가 적용

따라서 原価引き 수량은 항상 대응되는 定番 수량 이하가 된다.

**실제 판매수량 분해 방법:**

- 原価引き 수량이 N개일 때, 대응 定番 수량에서 N을 차감한 나머지가 「定番のみ適用」수량이다.
  - 定番만 적용 수량 = 定番 수량 − 原価引き 수량
  - 原価引き 적용 수량 = 原価引き 수량 (문서 기재값 그대로)
- 이 두 그룹의 합이 실제 판매수량이 된다.

예시 (管理No:1565564/1565565):

- 定番 264개 / 原価引き 204개
  → 60개(= 264 − 204) = 定番만 적용
  → 204개 = 定番 + 原価引き 동시 적용
  → **실제 판매수량 = 264개**

■ **導入条件**
신규 제품 입점·진열 시 발생하는 추가 조건이다.
導入조건은 대응되는 定番조건과 동일 판매수량으로 1:1 대응된다.

예시:

- 定番 6,672개 / 導入 6,672개
  → 전체 6,672개에 대해 定番 + 導入 조건이 동시에 적용

**주의사항:**

- 동일 상품에서 原価引き와 導入가 동시에 동일 수량으로 병렬 등장하는 경우는 일반적인 패턴이 아니다.
- 해당 경우는 OCR 판독 오류, 행 누락, 또는 조건 타입 오인 가능성을 우선 검토한다.

**Phase 2 처리 원칙:**

Phase 2에서는 조건 간 관계를 계산·합산하지 않는다.
동일 상품에 복수 管理No가 존재하더라도:

- 管理No 단위로 개별 row 추출
- 조건별 독립 item 유지
- 수량 merge / netting / 중복 제거 미수행

즉, 실제 판매수량 해석과 별개로 청구서 표기 구조 자체를 그대로 보존하여 추출한다.

## [Phase 2] OCR 정규화 규칙

이 양식의 테이블은 숫자 열을 **整数部 / 小数部** 두 서브컬럼으로 나눠 인쇄한다.
OCR은 두 서브컬럼을 구분자 없이 연결해서 읽기 때문에 아래와 같은 변환이 필요하다.

### 未収条件 — 小数点 복원 (÷100)

OCR이 읽은 값의 마지막 2자리가 소수부이므로 실제값 = OCR값 ÷ 100.

| OCR 값    | 실제 값    | 해석                  |
| --------- | ---------- | --------------------- |
| `13700` | `137.00` | 整数部=137, 小数部=00 |
| `1250`  | `12.50`  | 整数部=12, 小数部=50  |
| `4444`  | `44.44`  | 整数部=44, 小数部=44  |
| `4744`  | `47.44`  | 整数部=47, 小数部=44  |
| `370`   | `3.70`   | 整数部=3, 小数部=70   |

Phase 2 출력 시 `columns["未収条件"]`에는 **실제값(소수 포함)** 을 기록한다.
예: `137.00` (float)

### 対象数量 — 콤마 오독 정규화

수량의 천단위 구분자 `,`를 OCR이 `.` 또는 `:`로 잘못 읽는 경우가 있다.

| OCR 원문    | 실제 수량 |
| ----------- | --------- |
| `6.708個` | 6,708     |
| `8:256個` | 8,256     |
| `4.515個` | 4,515     |

정규화 규칙: 수량 내 `.` 또는 `:` → 제거 후 정수로 파싱.

Phase 2 출력 시 `columns["数量"]`에는 **정수**를 기록한다.
예: `6708` (int)

## [Phase 2] 함정 케이스

1. **집계 행 제외**: `＊ 管理No 計:` / `** 得意先 計:` / `*** 入出荷センター 計:` 행은 item이 아님 → 제외.
   OCR에 따라 전각 `＊` 대신 반각 `*`이 출력되는 경우가 있다. `* 管理No 計:`도 동일하게 집계 행으로 처리한다.
2. **계층 헤더 행 제외**: `計上場所:`, `入出荷支店:`, `入出荷センター:`, `得意先:` 행도 item 아님.
3. **得意先名 오인 주의**: 문서에 아래와 같이 두 줄이 연속으로 나타난다:
   ```
   入出荷センター：R北陸福井常温物流
   得意先　　　　：（株）ファミリーマート
   ```

   `R北陸福井常温物流`(물류센터명)는 **得意先이 아니다**. 得意先는 반드시 `（株）ファミリーマート`이다.- ❌ 잘못된 추출: 得意先名 =「R北陸福井常温物流」(물류센터명)
   - ✅ 올바른 추출: 得意先名 =「(株) ファミリーマート」
4. **お支払明細書**: `payment_form` role로 분류하고 Phase 2에서 일절 무시.
5. **備考 `※`**: 제품명 앞의 `※` 기호는 제품명에서 제거하고 추출.
6. **数量 소수 OCR**: 위 "OCR 정규화 규칙" 참조.
7. **得意先名 상속**: 동일 得意先 블록 안에서는 管理No마다 得意先名을 반복 기재한다.
8. **복수 請求書 in one PDF**: P-번호 또는 請求書No 변경으로 문서 경계 감지. 각 請求書No를 독립 문서로 처리.
9. **같은 제품명 반복 등장 시 누락 금지** (중요):
   같은 제품명이 **페이지 내·페이지 간** 여러 管理No 블록에 걸쳐 반복 등장하더라도, 각 管理No 블록의 모든 제품 행을 독립적으로 추출한다.
   앞서 같은 제품명·수량·금액 조합을 이미 추출했더라도, 管理No가 다르면 반드시 별개 item으로 출력한다.
   페이지가 다르더라도 동일하게 적용된다.
   - ❌ 잘못된 처리: "農心 辛ラーメントゥーンバカップ 113g / 数量204 / 金額2550은 管理No:1710186에서 이미 추출했으므로 管理No:1710201은 생략"
   - ✅ 올바른 처리: 管理No:1710186(page 5)과 管理No:1710201(page 7)은 수량·금액이 동일하더라도 서로 다른 item이므로 둘 다 추출
10. **소량·소액 행 누락 금지** (중요):
    수量이 적거나(예: 20個, 21個) 金額이 작더라도(예: 284円, 2877円) 절대 생략하지 않는다.
    原価引き条件 블록에는 구조적으로 소량·소액 행이 포함될 수 있다(対応する定番 수량의 일부에만 적용). 이를 오류 행이나 집계 행으로 오인하지 않는다.
    - ❌ 잘못된 처리: "金額 284円은 너무 작아 집계 노이즈로 판단, 추출 생략"
    - ✅ 올바른 처리: 管理No:XXXXXXX 原価引き条件 블록의 20個 284円 행도 그대로 item으로 추출
11. **入出荷支店 변경 직후 첫 번째 상품행 누락 금지** (중요):
    入出荷支店計·得意先計·入出荷センター計 집계 행들(4행) 이후 새 入出荷支店 섹션이 시작될 때, 4줄의 헤더(入出荷支店·入出荷センター·得意先·管理No)를 지난 직후에 상품 데이터 행이 온다.
    이 첫 번째 상품 행을 헤더의 연속으로 오인해 건너뛰는 패턴이 자주 발생한다.
    管理No 헤더 행 바로 다음 행은 반드시 item으로 추출한다.
    - ❌ 잘못된 처리: 入出荷支店 변경 후 管理No:1565567 첫 번째 상품(農心 辛ラーメン 120g×3, 金額=7398)을 헤더 행으로 오인해 생략, 두 번째 행(金額=1289)부터 추출
    - ✅ 올바른 처리: 管理No:1565567 블록의 첫 행인 農心 辛ラーメン 120g×3도 item으로 추출
12. **管理No 計: 역산 검증** (중요):
    각 管理No 블록 마지막의 `＊ 管理No 計:` 합계 金額을 해당 블록의 추출된 item 金額 합산과 비교한다.
    합산이 맞지 않으면 추출 누락이 발생한 것이므로 해당 블록을 재검토해 누락 행을 찾아 추가한다.

## [Phase 2] タイプ 분류 규칙

각 item의 タイプ를 아래 규칙 순서로 판단한다. 위에서 매칭되면 아래 규칙은 확인하지 않는다.
`condition_type`(`定番条件` / `原価引き条件` / `導入条件`)과 독립적으로 적용한다.

타입 9종은 모든 양식 공통 고정값. 양식마다 달라지는 것은 **조건 컬럼만**.

| 우선순위 | 조건                                                                                              | タイプ        | 제품매핑 | 비고                  |
| -------- | ------------------------------------------------------------------------------------------------- | ------------- | -------- | --------------------- |
| 1        | ※현업 확인 필요                                                                                  | `非課税`    | TBD      |                       |
| 2        | 商品名에 CF키워드(`センターフィ` `物流費` `配送協力金` `運賃` `役務`) 포함 + 同一請求書No의 cover `税率` = 8%계열 | `CF8%`      | TBD      | 센터피 항목 + 세율 8% |
| 3        | 商品名에 위 CF키워드 포함 + 同一請求書No의 cover `税率` = 10%계열                              | `CF10%`     | TBD      | 센터피 항목 + 세율 10% |
| 4        | 未収条件 = 공백(0/None) + CF키워드 없음 + 同一請求書No의 cover `税率` = 8%계열                 | `販促費8%`  | TBD      |                       |
| 5        | 未収条件 = 공백(0/None) + CF키워드 없음 + 同一請求書No의 cover `税率` = 10%계열               | `販促費10%` | TBD      |                       |
| 6        | ※현업 확인 필요                                                                                  | `ロットアウト` | TBD   |                       |
| 7        | ※현업 확인 필요                                                                                  | `消費税8%`  | TBD      |                       |
| 8        | ※현업 확인 필요                                                                                  | `消費税10%` | TBD      |                       |
| 9        | 이외 모든 항목 (기본값)                                                                           | `条件`      | ✅       |                       |

"8%계열"은 `8%`, `8.00%` 등 8로 시작하는 세율 표기를 모두 포함한다. "10%계열"은 `10%`, `10.00%` 등 10으로 시작하는 세율 표기를 모두 포함한다.

조건이 비어있는 규칙은 현업 확인 후 채워넣는다. 확인 전까지는 `条件`으로 출력한다.

**제품매핑 규칙**: `条件` 타입만 제품코드 매핑을 수행한다. `CF`·`販促費` 타입은 수수료 항목으로 실제 제품코드가 없으므로 매핑 불필요.

## 판매처 결정 규칙 (1:N 케이스)

소매처 하나에 판매처 후보가 여러 개일 때 아래 순서로 결정한다.

**1순위 — jisho(入出荷支店) 대조**

Phase 2가 추출한 items[]에서 해당 소매처가 속한 jisho 값을 읽는다.
jisho는 日本アクセスの 담당 영업부문(CVS営業部, R営業東北 등)을 직접 나타내므로
판매처를 특정하는 가장 신뢰할 수 있는 근거다.

대조 규칙:

- jisho = `CVS営業部` → **株式会社日本アクセス広域リテール営業本部加工食品飲料部** 로 확정
- jisho에 지역명이 포함된 경우(예: `R営業東北`, `R営業北陸`, `R営業中四国`, `R営業九州` 등)
  → 판매처 후보 목록에서 해당 지역명을 포함하는 판매처를 선택
  → 후보가 1건이면 자동 확정, 복수이면 2순위(issuer TEL 대조)로 진행

| 入出荷支店 패턴              | 해당 판매처                                                | 비고                                 |
| ---------------------------- | ---------------------------------------------------------- | ------------------------------------ |
| `CVS営業部`                | 株式会社日本アクセス広域リテール営業本部加工食品飲料部     | CVS専用営業部門                      |
| `R営業東北` 등 지역명 포함 | 地域名에 대응하는 판매처 후보 중 동일 지역명을 가진 판매처 | 예:`R営業東北` → 東北 담당 판매처 |
| 上記以外                     | 사용자 확인 요청                                           | 지역명 대조 불가 시                  |

**2순위 — issuer(발행처 TEL) 대조**

jisho로 특정 불가 시, cover 페이지의 TEL과 판매처 후보 명칭을 대조한다.

**3순위 — 사용자 확인 요청**

위 두 방법으로 특정 불가 시, 후보 목
