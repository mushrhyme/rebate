# 商談条件請求書 — tet

- **약칭**: tet
- **수신처**: (株)農心ジャパン
- **메모**: 伊藤忠食品株式会社 발행. 8%対象のみ確認済み。
- **특이사항**: 한 PDF = 단일 請求書 (cover 1枚 + detail 複数枚 + summary 1枚)

## 식별 패턴

`商談条件請求書` · `請求伝票番号` · `得意先名` · `ケース入数` · `条件` · `条件区分` · `消費税率`

---

## 문서 구조

```
PDF p.1      — cover  (請求書サマリー: 前回請求額合計 / 入金額 / 当月請求額 / 今回請求金額合計)
PDF p.2〜13  — detail (明細書: 請求伝票番号ごとの商品行)
PDF p.14     — summary (得意先別 小計一覧)
```

---

## 문서 내 계층 구조

```
請求書 (PDF全体)          — 추출 안 함
  └ 請求伝票番号 / 計上番号  → items[].invoice_no 로 상속
      └ 得意先名            → items[].customer 로 상속
          └ 商品行           ← 항목 추출 단위
```

## [Phase 2] 함정 케이스

1. **請求伝票番号小計行の除外**: 각 請求伝票番号グループの末尾に「請求伝票番号小計」行が存在する。この行は金額集計行であり、item として抽出しない。
2. **得意先小計行の除外**: summary ページに「得意先小計」行が存在する。同様に item として抽出しない。
3. **マイナス数量行の扱い**: 数量がマイナス（例: -12個）の行は返品・訂正行として存在する。item として抽出するが `数量` をマイナス値のまま保持する。
4. **計上ＮＯの複数行**: 同一請求伝票番号に対して計上ＮＯ（下段）が連番で複数存在する場合（例: `2-A8009-03`）、それぞれ別 item として扱う。
5. **得意先名の「集約口座」表記**: 得意先名に「《集」「集約口座」等の付記がある場合も customer フィールドに原文のまま含める。
6. **消費税率「外税」表記**: 消費税率列は「8.00% 外税」のような文字列。数値部分（8.00 または 10.00）のみ抽出して正規化する。
7. **Page 番号の連続性**: 明細書は Page 7〜13（またはそれ以上）に渡る。PDF の物理ページ番号と document 内 Page 番号が異なる場合があるため、ヘッダーの「Page:」値を優先する。

## [Phase 2] 크로스 페이지 감지

동일 請求伝票番号グループが 여러 페이지에 걸치는 경우 `source_pages`에 양 페이지 번호를 모두 포함한다.

- **단서 1**: 페이지 말미에 「請求伝票番号小計」행이 없고 次ページへの継続が示唆される場合 → 다음 페이지로 이어짐
- **단서 2**: 다음 페이지 상단の最初のグループが前ページ末尾と同一の「請求伝票番号」を持ち、小計行が存在しない場合 → 直前グループの연속
