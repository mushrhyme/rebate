# リベート合計表 — ヤマエ

- **약칭**: ヤマエ
- **수신처**: (株)農心ジャパン
- **메모**: リベート合計表 + リベート請求明細書 2페이지 구성
- **특이사항**: 한 PDF = cover(リベート合計表) + detail(リベート請求明細書) 구성. 합계表 p.1이 cover, 明細書 p.2가 detail.

## 식별 패턴

`リベート合計表` · `リベート請求明細書` · `リベート条件` · `リベート金額（税別）`

---

## 문서 구조

```
PDF p.1    — cover  (リベート合計表: 請求No, 得意先, 御入金予定日, 税抜金額, 消費税金額, 税込金額)
PDF p.2〜  — detail (リベート請求明細書: JANコード, 商品名, 取引数量合計, リベート条件, リベート金額)
```

---

## 문서 내 계층 구조

```
得意先様          → items[].customer 로 상속
  └ 明細行        ← 항목 추출 단위
```

## [Phase 2] 페이지 역할

| role      | 판단 기준                                              |
| --------- | ------------------------------------------------------ |
| `cover`   | 페이지 상단에 `リベート合計表` 텍스트가 존재하는 페이지 |
| `detail`  | 페이지 상단에 `リベート請求明細書` 텍스트가 존재하는 페이지 |

cover 페이지 totals 키:

| 키          | 필수 | 타입 | 비고                        |
| ----------- | ---- | ---- | --------------------------- |
| `税抜金額`  | ✓   | 정수 |                             |
| `消費税金額`| ✓   | 정수 |                             |
| `税込金額`  | ✓   | 정수 | 合計行 기준                 |

## [Phase 2] OCR 정규화 규칙

### リベート条件（単価）

| OCR 값       | 실제 값   | 해석                          |
| ------------ | --------- | ----------------------------- |
| `145.00円`   | 145.00    | 円 문자 제거 후 float 변환    |
| `45.00円`    | 45.00     | 円 문자 제거 후 float 변환    |

### 税率

| OCR 값   | 실제 값 | 해석                       |
| -------- | ------- | -------------------------- |
| `※8.0%` | 8       | ※ 및 % 제거 후 정수 변환  |
| `※10%`  | 10      | ※ 및 % 제거 후 정수 변환  |

## [Phase 2] 함정 케이스

1. **合計行 제외**: detail 페이지 하단 `合計` 행은 집계 행이므로 items 추출에서 제외한다.
2. **以下余白 / 以下空白 행 제외**: `以下余白` 또는 `以下空白` 텍스트가 포함된 행은 item으로 추출하지 않는다.
3. **cover의 明細行**: cover p.1의 【明細】テーブル 행(請求No, 契約期間, 請求締日, 件名, 税率, 税抜金額, 消費税金額, 税込金額)은 totals 보완 정보로만 활용하며, detail 페이지 명세와 중복 추출하지 않는다.
4. **税率の継承**: detail 페이지에 税率 열이 없는 경우 cover 明細행의 税率 값을 상속한다.
5. **請求No と 契約No の関係**: cover 明細의 `請求No（契約No）` 열 값을 invoice_no로 사용한다. 동일 PDF 내 복수 明細行이 존재할 경우 각 行별로 별도 item 그룹으로 분리한다.

## [Phase 2] 크로스 페이지 감지

동일 得意先 그룹이 여러 페이지에 걸치는 경우 `source_pages`에 양 페이지 번호를 모두 포함한다.

- **단서 1**: detail 페이지 말미에 `以下空白` 또는 `以下余白` 없이 페이지가 종료되는 경우 → 다음 페이지로 이어짐
- **단서 2**: 다음 detail 페이지 상단에 `得意先様` 헤더 없이 명세 행이 시작되는 경우 → 직전 그룹의 연속

## [Phase 2] タイプ 분류 규칙

각 item의 タイプ를 아래 규칙 순서로 판단한다. 위에서 매칭되면 아래는 확인하지 않는다.

타입 9종은 모든 양식 공통 고정값. 양식마다 달라지는 것은 **조건 컬럼만**.

| 우선순위 | 조건                    | タイプ         | 제품매핑 | 비고       |
| -------- | ----------------------- | -------------- | -------- | ---------- |
| 1        | TBD                     | `非課税`       | TBD      |            |
| 2        | TBD                     | `CF8%`         | TBD      |            |
| 3        | TBD                     | `CF10%`        | TBD      |            |
| 4        | TBD                     | `販促費8%`     | TBD      |            |
| 5        | TBD                     | `販促費10%`    | TBD      |            |
| 6        | TBD                     | `ロットアウト` | TBD      |            |
| 7        | TBD                     | `消費税8%`     | TBD      |            |
| 8        | TBD                     | `消費税10%`    | TBD      |            |
| 9        | 이외 모든 항목 (기본값) | `条件`         | —        | ✅ 확정    |

조건이 비어있는 규칙은 현업 확인 후 채워넣는다. 확인 전까지는 `条件`으로 출력한다.
