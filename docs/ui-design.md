# UI 화면 설계

> 현업(비개발자) 대상 피드백 문서.  
> 각 화면의 구현 상태와 미구현 사항을 함께 기술합니다.
>
> **범례**
> - ✅ 구현 완료 — 지금 바로 사용 가능
> - ⚠️ 부분 구현 — 화면은 있으나 일부 기능 미작동
> - 🔲 미구현 — 설계만 있고 아직 코드 없음
> - ❓ 미결정 — 업무규칙 또는 방향 확정 필요

## 0. 로그인 ✅

**진입 조건:** 미인증 상태에서 모든 페이지 접근 시 자동 리다이렉트

- 아이디 / 비밀번호 입력
- 로그인 성공 → 대시보드로 이동
- 세션은 localStorage의 `session_id`로 유지 (브라우저 닫아도 유지)
- 세션 만료 시 → 자동으로 로그인 페이지로 복귀

## 1. 대시보드 ✅

**목적:** 전체 처리 현황 파악 + 확인 대기 알림

### 상단 요약 카드 (3개)

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  확인 대기       │  │  처리 중         │  │  완료           │
│  2건 ↗          │  │  1건            │  │  12건           │
│  클릭해서 처리  │  │  자동 진행 중   │  │  이번 달 기준   │
└─────────────────┘  └─────────────────┘  └─────────────────┘
        ↑ 주황색 강조, 클릭 시 매핑 확인 화면으로 이동
```

### 문서 목록 테이블

| 문서명(doc_id) | 양식 | 상태 | 토큰 사용량 | 업로드일 | 액션 |
|---|---|---|---|---|---|
| 日本アクセスＣＶＳ | form_04 | 확인 대기 3건 | phase1 2.1k↑ 0.8k↓ / $0.003 | 05-13 | **확인하기** ↺ |
| 三菱食品東日本_2025_01 | form_01 | 완료 | phase1~3 합산 / $0.012 | 05-12 | **결과보기** ↺ |
| unknown_doc | — | 오류 | — | 05-11 | **오류 확인** |

**상태 배지:**
- `OCR중` / `분석중` → 파란 배지, 3초마다 자동 새로고침
- `확인 대기` → 주황 배지, 건수 표시
- `완료` → 초록 배지
- `오류` → 빨간 배지

**액션 버튼:**
- **확인하기** (확인 대기) → 매핑 확인 화면
- **결과보기** (완료) → 분석 결과 화면
- **오류 확인** (오류) → 오류 상세 모달 (아래 참조)

**오류 모달 (✅):**
```
[오류 유형 태그]  처리 오류
오류 내용: _______________

[재시도]  [신규 양식으로 등록 →]  ← 미인식 양식일 때만 표시
```
- 재시도 → 오류 문서를 처음부터 다시 분석 (OCR 캐시 유지 → Azure 비용 없음)
- 신규 양식 등록 → Cold-start 화면으로 이동

### ✅ 재분석 버튼 (완료/대기 상태)

완료·확인대기 상태 문서에 대한 재분석(강제 재수행)이 구현되었습니다.

문서 행 우측에 ↺ 아이콘 버튼이 추가되었습니다. 클릭 시 확인 다이얼로그가 표시되고, 승인하면 Phase 1~4를 처음부터 재실행합니다.  
재분석 시 OCR 캐시는 유지(Azure 재호출 없음), 기존 매핑 확인 내용은 초기화됩니다.

**참고 — 동일 파일 재업로드와 재분석의 차이:**
- **재업로드**: DB 상태만 초기화됨. 기존 매핑 확인 내용·분석 산출물이 남아 있어 이전 결과가 섞일 수 있습니다. **권장하지 않음.**
- **재분석 버튼**: 산출물·매핑을 모두 정리하고 처음부터 다시 돌립니다. **안전한 방법.**

## 2. 청구서 업로드 ✅

**목적:** PDF 파일을 업로드하고 분석을 시작

```
┌──────────────────────────────────────────────────┐
│                                                  │
│           ↑ PDF를 여기에 드래그하거나             │
│             파일을 선택하세요                     │
│                                                  │
│           PDF 파일만 허용됩니다                   │
└──────────────────────────────────────────────────┘

선택된 파일:
  📄 日本アクセスＣＶＳ.pdf      [×]
  📄 三菱食品東日本_2025.01.pdf  [×]

                        [분석 시작]
```

- 드래그앤드롭 또는 파일 선택 버튼
- 복수 파일 동시 선택 가능
- PDF 이외 파일은 자동 제외
- 분석 시작 클릭 → 백그라운드 처리 시작 → 대시보드로 자동 이동
- 파일명에서 doc_id 자동 생성 (괄호·공백 정리)

### ❓ 미결정: 동일 doc_id 중복 업로드 처리

같은 파일명의 PDF를 다시 올리면 기존 분석 결과가 덮어씌워집니다.  
"이미 존재합니다, 덮어쓸까요?" 확인 다이얼로그가 필요한지 여부를 검토 요청드립니다.

## 3. 매핑 확인 ✅

**목적:** Phase 3에서 자동 매핑에 실패한 항목을 현업이 직접 확정

**레이아웃: 좌우 2분할**

```
┌─────────────────────────────┬──────────────────────────────┐
│                             │  [문서탭1] [문서탭2]          │
│                             ├──────────────────────────────┤
│    좌: PDF 원본 뷰어        │  진행률 ████░░░░  2/5        │
│    (해당 페이지 표시)        ├──────────────────────────────┤
│    (OCR 인식 텍스트         │  소매처 매핑 1/3              │
│     오렌지 박스로 하이라이트)│                              │
│                             │  OCR 원문:                   │
│    ◄  페이지 2/14  ►        │  ローソントウカイ              │
│                             │                              │
│                             │  ● ローソン東海 (1991474) ✓  │
│                             │  ○ ローソン東海北 (1991475)  │
│                             │  ○ ローソン東海南 (1991476)  │
│                             │                              │
│                             │  [직접 검색]                 │
│                             │                              │
│                             │  [확정하고 캐시에 저장]       │
└─────────────────────────────┴──────────────────────────────┘
```

### 우측 패널 세부 동작

**문서 탭 (✅):**  
확인 대기 중인 문서가 여러 개면 탭으로 전환 가능. 탭마다 미확정 건수 표시.

**진행률 바 (✅):**  
전체 미확정 건수 대비 확정 완료 건수 실시간 표시.

**매핑 카드 항목별 동작:**
- **후보 목록 있는 경우 (✅):** 점수 높은 순으로 후보 표시. 라디오 선택 후 확정.
- **후보 없는 경우 (✅):** OCR 명칭으로 자동 검색 후 결과 표시. 결과도 없으면 경고 메시지.
- **직접 검색 (✅):** "직접 검색" 버튼 → 검색 모달 팝업 → 입력하면 실시간 검색.

**확정 버튼 (✅):**
- 클릭 시 선택 항목 서버에 저장 + 캐시(CSV) 자동 누적
- 모든 항목 확정 시 → Phase 4 NET 계산 자동 시작

### ✅ PDF 연동

**동작:**
- 매핑 항목 선택 시 → 해당 항목이 등장한 페이지로 자동 이동
- 해당 OCR 텍스트를 오렌지 박스로 하이라이트

**수정 완료 사항:**
- `page_number` 필드명 불일치 버그(`"page"` vs `"source_pages"`) 수정 완료
- `.ocr.json` 생성 로직 추가 완료 (`_azure_raw.json` 저장 포함)
- 신규 분석 문서는 하이라이트 + 페이지 자동 이동 모두 정상 동작

**제약 조건:**
- 기존 분석 문서(이번 업데이트 이전 분석분)는 `.ocr.json` 파일이 없어 PDF 연동이 동작하지 않음
  → **조치:** 해당 문서를 재분석(↺ 버튼)하면 해결

### 교차검증 배지 (✅)

Phase 4가 PDF 표지·합계 행과 계산 결과를 자동으로 비교합니다.
- ✅ 일치 → 초록 배지
- ⚠️ 불일치 → 주황 배지 (기준값 함께 표시)

### 결과 테이블 (✅)

- 거래처(OCR 원문)별로 그룹화
- 그룹 헤더: 소매처코드·명, 판매처코드·명, 소계
- 그룹 클릭 → 좌측 PDF에서 해당 거래처명 하이라이트
- 행 내 경고 (⚠️): `unconfirmed`(미확정 매핑 포함) 또는 `net_lt_honbu`(NET < 본부장)

### 🔲 미구현: 내보내기

현재 "내보내기" 버튼은 UI만 존재하며 기능이 없습니다.

> **미결정 사항:** 어떤 포맷으로 내보낼지 확정 필요  
> 후보: Excel / CSV / 직접 DB 업로드 (`upload_to_db.py`)

### ⚠️ 부분 구현: PDF ↔ 결과 연동

- 결과 행 클릭 → OCR 텍스트 하이라이트는 동작 (`.ocr.json` 파일이 있는 경우)
- 행 클릭 시 해당 페이지로 자동 이동은 **미구현** (Phase4Row에 페이지 번호가 없음)
  - 사용자가 직접 PDF 페이지를 앞뒤로 넘기며 확인해야 함

## 5. Form 관리 ⚠️

**목적:** 등록된 양식 목록 확인 + 규칙 업데이트

**레이아웃: 좌우 2분할**

```
┌──────────────────┬───────────────────────────────────────────┐
│ 양식 목록        │  form_01 · FINET 買掛金特別値引           │
│                  │  TBD 항목: 2건                            │
│ form_01  TBD 2  │  ─────────────────────────────────────── │
│ form_04  TBD 0  │  (form 정의 MD 전문 표시)                 │
│                  │                                           │
│ [신규 양식 등록] │  ─────────────────────────────────────── │
│                  │  💬 대화로 규칙 수정                      │
│                  │  [이미지 첨부] [메시지 입력...]  [전송]   │
│                  │                                           │
│                  │  Claude: form_01의 CF 조건 분류 기준을... │
└──────────────────┴───────────────────────────────────────────┘
```

### 구현된 기능 (✅)

- 좌측: 등록된 양식 목록, TBD 건수 표시
- 우측 상단: 선택된 양식의 form 정의 전문 표시
- 우측 하단: 채팅 인터페이스 (텍스트 + 이미지 첨부)

### ⚠️ 미구현: 실제 저장 연결

현재 채팅 UI는 화면만 있고, **Claude API 연결 및 form 파일 실제 저장이 미구현**입니다.

> **개발 필요:**  
> 1. 채팅 메시지 → 백엔드 API → Claude 호출 → form 파일 수정  
> 2. 이미지 첨부 → update-form 스킬 호출 → 변경 내용 미리보기 → 확정 저장

### 🔲 미구현: TBD 항목 관리

- TBD 항목 클릭 → 인라인 입력으로 규칙 확정 UI 없음
- 신규 양식 등록 버튼 → Cold-start 화면 연결만 됨 (상세 UI 없음)

## 6. 신규 양식 등록 (Cold-start) ⚠️

**목적:** 처음 보는 양식의 form 정의를 대화형으로 작성

**진입 경로:**
- 대시보드 오류 모달에서 "신규 양식으로 등록" 클릭 (양식 미인식 오류 시)
- Form 관리 화면에서 "신규 양식 등록" 클릭

```
┌───────────────────────────────────────────────────────────┐
│  신규 양식 등록                                           │
│  처음 보는 청구서 양식의 규칙을 함께 만들어봅니다         │
├───────────────────────────────────────────────────────────┤
│                                                           │
│  이 문서의 발행처는 어느 회사인가요?                      │
│                                                           │
│  어떤 열(컬럼)들이 있나요?                                │
│  PDF 이미지를 첨부해주시면 제가 분석해드릴게요.           │
│                                                           │
│  ─────────────────────────────────────────────────────── │
│  💬 대화 입력창                                           │
│  [이미지 첨부] [입력...]  [전송]                          │
└───────────────────────────────────────────────────────────┘
```

### ⚠️ 현재 상태

대화형 UI는 존재하지만, **실제 저장 및 파이프라인 연결이 미구현**입니다.

> **개발 필요:**  
> 1. 채팅 입력 → Claude API → form 정의 초안 자동 생성  
> 2. 초안 미리보기 → 현업 확인 → 저장  
> 3. 저장 후 → 해당 문서 자동 재분석 시작

**Cold-start 단계 설계 (목표):**

| 단계 | 내용 | 현업 입력 |
|------|------|-----------|
| Step 1 | 발행처명 확인 | 텍스트 입력 |
| Step 2 | 컬럼 구성 파악 | PDF 이미지 첨부 → Claude가 자동 추출 → 현업 확인 |
| Step 3 | 타입 분류 규칙 | CF 키워드, NET 계산식 입력 (모르면 "나중에" → TBD 저장) |
| Step 4 | 최종 확인·저장 | form 정의 미리보기 → 승인 |
