# 시스템 구조 — 분석 레이어와 가이드 레이어

## 핵심 개념

이 시스템은 두 레이어가 분리되어 있다.

```
┌─────────────────────────────────────────────────────┐
│  가이드 레이어 (현업이 소유)                           │
│                                                      │
│  form_definitions/form_XX.md  ← 여기가 지능의 핵심   │
│  mappings/*.csv                                      │
│  form_definitions/image/                             │
└────────────────────┬────────────────────────────────┘
                     │ 읽는다
┌────────────────────▼────────────────────────────────┐
│  분석 레이어 (개발자가 소유)                           │
│                                                      │
│  Phase 1 → 2 → 3 → 4 파이프라인                      │
│  (.claude/skills, agents, scripts/)                  │
└─────────────────────────────────────────────────────┘
```

**분석 레이어**는 고정된 파이프라인이다. 코드다. 개발자가 만들고 유지한다.  
**가이드 레이어**는 파이프라인에게 "이 양식은 이렇게 읽어라"를 가르치는 설정이다. MD 파일이다. 현업이 소유하고 확장한다.

---

## 왜 이렇게 나누는가

기존 시스템의 문제: 새 양식이 들어오면 개발자가 코드를 고쳐야 했다.  
새 시스템의 답: 새 양식이 들어오면 **현업이 MD 파일을 추가**하면 파이프라인이 즉시 그 양식을 처리할 수 있다.

코드 배포 없음. 개발자 연락 없음. form_XX.md 한 파일이 시스템의 지능을 확장한다.

---

## form_XX.md가 시스템에서 하는 역할

| Phase | form_XX.md에서 읽는 것 |
|-------|----------------------|
| Phase 1 | (영향 없음 — OCR 구조화) |
| Phase 2 | 추출 컬럼 정의, 페이지 역할, 함정 케이스, 출력 예시 |
| Phase 3 | タイプ 분류 규칙 |
| Phase 4 | NET 계산식, 교차검증 기준 |

form_XX.md가 없으면 파이프라인은 그 양식을 처리하지 못한다.  
form_XX.md가 있으면 파이프라인은 즉시 처리를 시작한다.

## 마중물 단계 — 신규 양식 대응 흐름

> "마중물": 펌프가 물을 올리려면 먼저 한 바가지를 부어야 한다.  
> form_XX.md 한 파일이 그 역할을 한다.

```
새 양식 PDF 도착
     │
     ▼
analyze-invoice 실행
     │
     ▼
Phase 2: 양식 식별 시도
     │
     ├─ 매칭 성공 ──────────────────────────► 정상 분석 진행
     │
     └─ 매칭 실패 (신규 양식)
          │
          ▼
     [현업 개입 — 여기가 마중물 단계]
          │
          ├─ 방법 A: 샘플 이미지 캡처 → 채팅에 붙여넣기 → "반영해줘"
          │          → update-form 스킬이 form_XX.md 초안 생성
          │
          └─ 방법 B: /cold-start 실행
                     → Claude와 대화로 form_XX.md 작성
          │
          ▼
     form_XX.md 사용자 승인 → _index.md 등록
          │
          ▼
     파이프라인 즉시 재개 ─────────────────► Phase 2 항목 추출
                                             Phase 3 매핑
                                             Phase 4 계산
                                             결과 표시
```

이 흐름이 끝나면 다음 번에 같은 양식이 들어왔을 때는 현업 개입 없이 자동으로 처리된다.
